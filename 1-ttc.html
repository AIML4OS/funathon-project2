Supervised approach using TorchTextClassifiers

This notebook walks through a supervised text classification pipeline using torchTextClassifiers, applied to NAF 2025 code prediction from synthetic labelled data.

The goal is to automatically assign a NAF activity code to a short text description of a business activity. This is a multiclass classification problem: given a label (a sentence like “I sell croissants and pains au chocolat”), the model must predict the correct code among hundreds of possible NAF categories.

1 Load data

The dataset consists of synthetic labelled examples generated for the NAF 2025 nomenclature. Each row contains a short text description (label) and the corresponding NAF code (code).

2 Split the data

Before training a model, we need to split the dataset into three subsets:

  • Train: the examples the model learns from.
  • Validation: used during training to monitor performance and trigger early stopping (i.e. stop training before overfitting).
  • Test: held out until the very end to give an unbiased estimate of the model’s performance on unseen data.
NoteWhy three splits and not two?

Using the validation set to tune hyperparameters or stop training means it indirectly influences the model. The test set must therefore remain completely untouched during training so that the final evaluation is truly unbiased.

def train_val_test_split(
    df: pl.DataFrame,
    train_frac: float = 0.7,
    val_frac: float = 0.15,
    seed: int = 42,
) -> tuple[pl.DataFrame, pl.DataFrame, pl.DataFrame]:
    df = df.sample(fraction=1.0, shuffle=True, seed=seed)
    n = len(df)
    n_train = int(n * train_frac)
    n_val = int(n * val_frac)
    train = df[:n_train]
    val   = df[n_train : n_train + n_val]
    test  = df[n_train + n_val :]
    return train, val, test

3 Tokenizer

Neural networks cannot process raw text directly — they work with numbers. A tokenizer is the component that bridges this gap: it splits a text into small units called tokens (words, subwords, or characters) and maps each token to a unique integer ID.

For example, the sentence “I sell croissants and pains au chocolat” might be split into tokens ["i", "sell'", "croissants", "and", "pains", "au", "chocolat"] and encoded as [326, 40, 199, 126, 20, 755, 1024].

The sequence of IDs is then padded or truncated to a fixed length (output_dim) so that all inputs have the same shape, which is required for batched training.

NoteWhat is WordPiece?

WordPiece is a subword tokenization algorithm. Rather than splitting on spaces (word-level) or individual characters, it finds a vocabulary of frequent subword units that can represent any word — including rare or unseen ones. For instance, “informatiques” might be split into ["informat", "##iques"]. The ## prefix indicates that the token is a continuation of the previous one. Because the vocabulary is built from your training corpus, it adapts well to domain-specific text like French business descriptions.

4 Model architecture

A text classification model in torchTextClassifiers is built from three components:

  1. TextEmbedder: converts token IDs into dense vectors (embeddings) of size embedding_dim. A self-attention layer can be added to capture word order and context.

  2. CategoricalVariableNet (optional): encodes additional categorical features and merges them with the text representation. Not used in this dataset.

  3. ClassificationHead: projects the final representation onto num_classes dimensions. The highest value determines the predicted class.

These are configured through ModelConfig and assembled automatically by torchTextClassifiers.

5 Training

Training a neural network means finding the weights that minimise a loss function — a measure of how wrong the model’s predictions are. At each step, the model makes predictions on a mini-batch of examples, computes the loss, and updates the weights in the direction that reduces it (via gradient descent).

Key training hyperparameters:

  • num_epochs: how many times the model sees the full training set.
  • batch_size: how many examples are processed at once before a weight update.
  • lr (learning rate): how large each weight update step is.
  • patience_early_stopping: stop training if the validation loss has not improved for this many epochs, to avoid overfitting.

import json
import torch
from transformers import PreTrainedTokenizerFast

MLFLOW_TRACKING_URI = os.environ["MLFLOW_TRACKING_URI"]

mlflow.set_tracking_uri(MLFLOW_TRACKING_URI)

artifacts_path = mlflow.artifacts.download_artifacts(f"runs:/{RUN_ID}/model_artifacts")

with open(os.path.join(artifacts_path, "model_config.json")) as f:
    saved_config = json.load(f)

with open(os.path.join(artifacts_path, "label_mapping.json")) as f:
    saved_label_mapping = json.load(f)  # {str(int): NAF code}

loaded_tokenizer = WordPieceTokenizer(
    vocab_size=saved_config["vocab_size"],
    output_dim=saved_config["output_dim"],
)
loaded_tokenizer.tokenizer = PreTrainedTokenizerFast.from_pretrained(
    os.path.join(artifacts_path, "tokenizer")
)
loaded_tokenizer.trained = True

loaded_model_config = ModelConfig(
    embedding_dim=saved_config["embedding_dim"],
    num_classes=saved_config["num_classes"],
    attention_config=AttentionConfig(
        n_layers=saved_config["n_layers"],
        n_head=saved_config["n_head"],
        n_kv_head=saved_config["n_kv_head"],
        sequence_len=saved_config["output_dim"],
    ),
    label_attention_config=LabelAttentionConfig(
        n_head=saved_config["label_attention_n_head"],
        num_classes=saved_config["num_classes"],
    ),
)

ttc_full = torchTextClassifiers(tokenizer=loaded_tokenizer, model_config=loaded_model_config)
ttc_full.pytorch_model.load_state_dict(
    torch.load(os.path.join(artifacts_path, "model_state_dict.pt"), weights_only=True)

ttc_full.pytorch_model.eval()
print("Pretrained model loaded successfully.")

from torchTextClassifiers.utilities.plot_explainability import (
    map_attributions_to_char, map_attributions_to_word,
    plot_attributions_at_char, plot_attributions_at_word, figshow,
)

text_idx = 0
top_k_idx = 0
text_sample         = example_texts[text_idx]
offsets             = results["offset_mapping"][text_idx]
word_ids            = results["word_ids"][text_idx]
predicted_class_idx = results["prediction"][text_idx][top_k_idx].item()
predicted_code      = saved_label_mapping[str(predicted_class_idx)]

# Average attention over heads, pick the row for the predicted class
attn_matrix  = results["label_attention_attributions"][text_idx].detach()  # (n_head, num_classes, seq_len)
attributions = attn_matrix.mean(dim=0)[predicted_class_idx]                # (seq_len,)

words, word_attributions = map_attributions_to_word(
    attributions.unsqueeze(0), text_sample, word_ids, offsets
)
char_attributions = map_attributions_to_char(attributions.unsqueeze(0), offsets, text_sample)

titles = [f"Attributions for NAF code {predicted_code}"]

figshow(plot_attributions_at_char(
    text=text_sample, attributions_per_char=char_attributions, titles=titles,
)[0])

figshow(plot_attributions_at_word(
    text=text_sample, words=words.values(), attributions_per_word=word_attributions, titles=titles,
)[0])
